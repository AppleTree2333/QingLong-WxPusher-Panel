# QingLong WxPusher Panel

这套方案用于给青龙挂机脚本提供统一的 `WxPusher HTML` 推送能力。

核心目标：

- 统一推送入口，不再让每个脚本自己拼 `requests.post(...)`
- 统一数据结构，让脚本作者只关心业务结果
- 统一前端面板，让消息在手机端更简洁、更好读
- 保留扩展能力，支持后续增加任务明细和补充信息

## 适用场景

- 多账号签到
- 多账号积分获取
- 多账号任务执行
- 需要汇总结果 + 单账号详情 + 少量明细的挂机脚本

## 青龙面板部署说明

如果你在青龙面板中使用本仓库脚本，请务必遵循以下文件放置规则：

1. **放置公共依赖**
   - 必须将 `wxpusher_utils.py` 下载并放置到青龙面板的 `scripts` 目录中。
   - 这是所有脚本共享的核心推送引擎。
2. **放置业务脚本**
   - 如果业务脚本放在 `scripts` 根目录，直接 `import wxpusher_utils`。
   - 如果业务脚本和配置文件放在**二级或三级子目录**（如 `scripts/my_project/`），必须在脚本头部动态注入 `sys.path`，以确保能找到根目录的公共引擎：
   ```python
   import os
   import sys

   # 动态将青龙 scripts 根目录加入模块搜索路径，兼容子目录调用
   _ql_scripts_dir = os.path.abspath(os.path.join(os.path.dirname(__file__), '..'))
   if _ql_scripts_dir not in sys.path:
       sys.path.append(_ql_scripts_dir)

   from wxpusher_utils import push_html_panel
   ```

## 环境变量

必须配置以下变量：

- `WXPUSHER_APP_TOKEN`
- `WXPUSHER_UID`

说明：

- `WXPUSHER_UID` 支持单个 UID
- 也支持逗号分隔的多个 UID

## 当前能力

- 统一发送函数：`send_wxpusher()`
- 统一 HTML 面板：`build_html_panel()`
- 统一正式推送入口：`push_html_panel()`
- **极简科技风面板**：采用现代化的间距、排版与微阴影卡片设计。
- 原生主题支持：`light` / `dark` / `auto` （支持系统深色模式自适应）。
- 可扩展数据结构：
  - `PanelMetric`
  - `PanelItem`
  - `PanelTag`
  - `PanelTask`
  - `PanelExtra`

## 最小用法

```python
from wxpusher_utils import PanelItem, PanelMetric, PanelTag, push_html_panel

metrics = [
    PanelMetric("总账号", "6", "primary"),
    PanelMetric("成功", "5", "success"),
    PanelMetric("失败", "1", "danger"),
]

items = [
    PanelItem(
        title="账号1",
        value="+12",
        desc="签到成功，积分已增加",
        meta="当前积分 348",
        status="success",
        tags=[PanelTag("签到成功", "success")],
    )
]

ok, data, html = push_html_panel(
    summary="今日任务完成 5 正常 / 1 异常",
    title="青龙任务日报",
    subtitle="多账号执行结果",
    metrics=metrics,
    items=items,
    footer="Generated by QingLongScript",
    theme="auto",
)
```

## 数据设计

整体遵循以下层级：

- 全局层：`summary`
- 概览层：`metrics`
- 账号层：`items`
- 任务层：`tasks`
- 附加层：`extras`

这样做的好处是：

- 用户先看总结果
- 再定位异常账号
- 需要时再看单账号任务明细
- 后续想补充余额、等级、奖励等字段时也有明确位置

## 主题机制

当前渲染器支持：

- `light`：浅色主题
- `dark`：深色主题
- `auto`：自动跟随系统色彩偏好

如果后续要换前端风格，只需要调整公共渲染器，不需要改业务脚本的数据结构。

## 推荐接入方式

后续所有正式推送脚本统一遵循：

- 业务脚本只负责采集和整理结构化结果
- 正式推送只调用 `push_html_panel()`
- 不在脚本中单独手写完整 HTML
- 不在脚本中直接调用 `WxPusher` HTTP 接口

## 配套文件

- `wxpusher_utils.py`：公共模块
- `WXPUSHER_AI_GUIDE.md`：面向 AI 的接入标准
- `WXPUSHER_TEMPLATE.py`：最小调用模板
