# WXPusher AI Guide

本规范用于指导其他 AI 为青龙 Python 脚本接入统一的 `WxPusher HTML` 推送。

目标只有一个：让 AI 不需要完整看完旧脚本去猜推送结构，而是直接按协议输出。

## 1. 目标范围

本规范适用于：

- 多账号挂机脚本
- 多账号签到脚本
- 多账号积分脚本
- 需要单次汇总推送的青龙任务

本规范不适用于：

- 单纯的控制台日志输出
- 纯调试脚本
- 必须使用其他通知渠道的场景

## 2. 强制规则

- 正式推送必须使用 `wxpusher_utils.py`
- 正式推送必须调用 `push_html_panel()`
- 禁止在业务脚本中直接 `requests.post(...)` 推送到 `WxPusher`
- 禁止在业务脚本中手写完整 HTML 面板
- 正式推送统一使用 `HTML`
- 单账号异常不得中断整批执行
- 业务脚本只负责产出结构化结果，不负责设计推送样式

## 3. 环境变量

必须使用以下变量名：

- `WXPUSHER_APP_TOKEN`
- `WXPUSHER_UID`

约束：

- `WXPUSHER_UID` 可为单个 UID
- 也可为逗号分隔的多个 UID
- 禁止自创第二套别名变量

## 4. 数据协议

### 4.1 顶层调用

必须调用：

```python
ok, data, html = push_html_panel(
    summary=summary,
    title=title,
    subtitle=subtitle,
    metrics=metrics,
    items=items,
    footer=footer,
    theme="auto",
)
```

### 4.2 status / tone 允许值

只允许以下值：

- `primary`
- `success`
- `warning`
- `danger`
- `neutral`

推荐映射：

- 成功：`success`
- 已完成但无变化：`neutral`
- 待处理 / 部分成功：`warning`
- 请求失败 / 鉴权失败 / 执行失败：`danger`

### 4.3 PanelMetric

用途：全局概览指标

字段：

- `label`
- `value`
- `tone`

规则：

- 建议 2 到 4 个
- 不要超过 4 个
- 典型字段：总账号、成功、失败、待处理

### 4.4 PanelItem

用途：单账号或单对象主卡片

字段：

- `title`
- `value`
- `desc`
- `meta`
- `status`
- `tags`
- `lines`
- `tasks`
- `extras`

说明：

- `title`：账号名或对象名
- `value`：主结论，如 `+12`、`成功`、`失败`
- `desc`：一句话摘要
- `meta`：次级信息
- `tags`：短标签
- `lines`：少量补充说明
- `tasks`：任务级明细
- `extras`：补充结构化数据

### 4.5 PanelTask

用途：单账号下的任务级详情

字段：

- `title`
- `status`
- `detail`
- `value`

适用场景：

- 想展示某个账号下每个任务的完成情况
- 想保留“任务名 + 状态 + 一句话结果”

### 4.6 PanelExtra

用途：附加结构化信息

字段：

- `label`
- `value`
- `tone`

适用场景：

- 余额
- 连续天数
- 等级
- 奖励数量
- 其他补充信息

## 5. 信息流规则

顺序固定为：

- `summary`
- `metrics`
- `items`
- `footer`

每个 `item` 内部推荐顺序：

- 标题与主结果
- 摘要说明
- 次级信息
- 补充信息 `extras`
- 任务明细 `tasks`
- 补充说明 `lines`
- 状态标签 `tags`

总原则：

- 先结论
- 后异常
- 再补充
- 禁止日志墙

## 6. 主题与渲染规则

允许值：

- `light`
- `dark`
- `auto`

默认推荐：

- `theme="auto"`

说明：

- 业务脚本不要自己管理主题色。
- 主题切换交给公共渲染器。
- **高级感设计约束**：
  - 前端面板已重构为现代化极简科技风，采用了 CSS 变量、Grid 布局、弥散阴影和细边框设计。
  - AI 在扩展渲染器时，**禁止使用行内硬编码颜色（如 `color: #ff0000`）**，必须使用 `_THEME_PALETTES` 中定义的 CSS 变量（如 `var(--t-danger-tx)`）。
  - 禁止使用过大的圆角（如 `border-radius: 999px` 仅限于标签，卡片使用 `16px`）。
  - 维持 `-apple-system, SF Pro Text` 的字体栈优先级。

## 7. 异常处理规则

- 单账号请求异常必须收敛成账号级结果
- 查询失败、任务失败、鉴权失败必须区分
- 不允许把所有异常都写成“任务不存在”或“请求失败”
- 推送失败要打印返回信息
- 多 UID 发送结果由公共模块统一校验

## 8. 禁止事项

- 禁止新增脚本内独立 HTML 模板
- 禁止新增脚本内独立 `WxPusher` 请求封装
- 禁止把完整日志塞进 `desc`
- 禁止把复杂结构全部硬塞进 `lines`
- 禁止绕过公共模块发送正式推送

## 9. 子目录调用规范（青龙面板）

如果脚本需要连同账号配置一起放入二级甚至三级文件夹（例如 `scripts/my_project/task.py`），为了确保能调用到根目录的 `wxpusher_utils.py`，**必须**在头部动态注入 `sys.path`：

```python
import os
import sys

# 获取当前脚本所在目录，向上寻找到 scripts 根目录并加入 sys.path
_ql_scripts_dir = os.path.abspath(os.path.join(os.path.dirname(__file__), '..'))
if _ql_scripts_dir not in sys.path:
    sys.path.append(_ql_scripts_dir)

from wxpusher_utils import push_html_panel
```

## 10. 最小模板

```python
from datetime import datetime
from wxpusher_utils import PanelExtra, PanelItem, PanelMetric, PanelTag, PanelTask, push_html_panel

results = [
    {
        "name": "账号1",
        "status": "success",
        "value": "+12",
        "desc": "任务全部完成",
        "meta": "当前积分 348",
        "tasks": [
            {"title": "签到", "status": "success", "detail": "已完成", "value": "+2"},
            {"title": "浏览", "status": "success", "detail": "已完成", "value": "+10"},
        ],
        "extras": [
            {"label": "连续天数", "value": "7", "tone": "primary"},
        ],
        "tags": [
            {"text": "全部完成", "tone": "success"},
        ],
    }
]

metrics = [
    PanelMetric("总账号", str(len(results)), "primary"),
    PanelMetric("成功", "1", "success"),
]

items = [
    PanelItem(
        title=r["name"],
        value=r["value"],
        desc=r["desc"],
        meta=r["meta"],
        status=r["status"],
        tasks=[PanelTask(**task) for task in r.get("tasks", [])],
        extras=[PanelExtra(**extra) for extra in r.get("extras", [])],
        tags=[PanelTag(**tag) for tag in r.get("tags", [])],
    )
    for r in results
]

ok, data, html = push_html_panel(
    summary="任务完成 1 正常 / 0 异常",
    title="脚本执行报告",
    subtitle=datetime.now().strftime("%Y-%m-%d %H:%M"),
    metrics=metrics,
    items=items,
    footer="Generated by script.py",
    theme="auto",
)
```

## 10. 接入验收清单

新脚本接入前必须自查：

- 是否只使用了 `push_html_panel()`
- 是否没有直接请求 `WxPusher API`
- 是否没有手写完整 HTML 模板
- 是否把单账号异常收敛为单独结果项
- 是否正确使用 `status/tone`
- 是否控制 `metrics` 在 4 个以内
- 是否只保留必要的 `lines`
- 是否真实跑过一次
- 是否确认推送消息可读且信息流简洁
